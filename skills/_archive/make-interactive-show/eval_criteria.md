# 互动影游剧本评估标准

> 16 个评估维度，分 4 大类。每个维度 1-10 分。
> 总分 160 分（含加分项 170）。及格线：总分 ≥ 112（70%），且无任何维度 ≤ 3。

---

## A. 角色设计 — 3 个维度

### A1. 人物与角色关系设定

**考核内容**: 主要角色是否立体鲜明？角色关系网是否丰富且有张力？

| 分数 | 标准 |
|------|------|
| 9-10 | 主角有明确的性格缺陷与优点并存；配角各有独特动机和行为逻辑；角色关系网包含至少 3 种张力类型（对立、依赖、暗恋、师徒、仇敌等）；角色间有"化学反应"——互动能产生意想不到的火花 |
| 7-8 | 主角性格鲜明；配角有区分度；角色关系有 2 种以上张力类型；角色互动合理自然 |
| 5-6 | 主角有基本性格；配角存在脸谱化倾向；角色关系单一（只有敌友二分）|
| 3-4 | 角色缺乏个性，互换名字不影响剧情；关系扁平 |
| 1-2 | 角色是功能性符号，无任何性格深度 |

**检查点**:
- [ ] 主角有 ≥ 3 个通过行为体现（而非标签告知）的性格特征
- [ ] 反派/对手有合理动机，不是"纯粹的恶"
- [ ] 配角至少有 1 个"意外行为"——做了与第一印象不同的事
- [ ] 角色关系图中存在至少 1 组三角关系（不限感情）
- [ ] 原作角色的核心特质在改编中被保留

---

### A2. 人物一致性（言行举止）

**考核内容**: 角色的言行举止是否前后统一？性格、身份、能力是否自洽？

| 分数 | 标准 |
|------|------|
| 9-10 | 全剧零人设崩塌；角色在不同分支中的行为差异可由"玩家选择导致的情境变化"合理解释；语言风格（用词、口头禅、说话方式）前后一致 |
| 7-8 | 无明显人设崩塌；偶有 1-2 处轻微不一致但不影响整体 |
| 5-6 | 出现 1-2 次明显的人设矛盾（如怯懦角色突然勇猛但无铺垫）|
| 3-4 | 多处人设崩塌；角色行为逻辑混乱 |
| 1-2 | 角色行为完全随机，无任何一致性可言 |

**检查点**:
- [ ] 同一角色在不同集/分支中的称谓一致
- [ ] 角色的知识水平一致（不会突然知道不该知道的事）
- [ ] 角色的能力水平一致（除非有合理的成长/削弱铺垫）
- [ ] 角色的语言风格一致（粗人不会突然文绉绉，书生不会突然粗口）
- [ ] 分支剧情中，角色在"同一时间点"的不同路线中性格基底相同
- [ ] 角色使用的能力/武器必须在获取/学习之后才使用
- [ ] 已死亡角色不得在死亡集之后出场
- [ ] 新角色首次出场必须有介绍场景（台词或舞台指示交代身份），不可凭空出现

---

### A3. 人物成长弧光

**考核内容**: 主要角色是否有可感知的成长或变化？人物弧是否有说服力？

| 分数 | 标准 |
|------|------|
| 9-10 | 主角有完整的"信念—挑战—改变"弧光；成长通过具体事件驱动而非旁白告知；至少 1 个配角也有可见弧光；不同分支/结局中弧光呈现差异化 |
| 7-8 | 主角弧光清晰；成长节点有事件支撑；配角有一定变化 |
| 5-6 | 主角有成长但靠旁白/他人评价表现而非行为转变；配角无变化 |
| 3-4 | 角色从头到尾无变化，是静态的 |
| 1-2 | 角色发展违反逻辑（如无缘无故黑化/洗白）|

**检查点**:
- [ ] 主角在 EP01 和最终集中面对相似情境时，反应明显不同（体现成长）
- [ ] 成长的转折点有具体场景承载而非一笔带过
- [ ] 成长是渐进的，不是突然的顿悟
- [ ] 不同结局中角色的最终状态反映了不同选择路径带来的不同成长方向

---

## B. 台词质量 — 2 个维度

### B1. 台词内容质量

**考核内容**: 台词是否有信息量、有个性、有推进力？

| 分数 | 标准 |
|------|------|
| 9-10 | 台词承担叙事功能（推进剧情、暗示伏笔、展示世界观）；每个角色的说话方式有辨识度；存在"金句"——读后能记住的台词；对白有潜台词层（表面说一件事，实际暗示另一件事）|
| 7-8 | 台词自然流畅，有角色区分度；重要场景有记忆点台词 |
| 5-6 | 台词完成基本功能但偏平淡；角色说话方式缺乏区分 |
| 3-4 | 台词生硬、口号化或过度解释 |
| 1-2 | 台词语病严重或与场景完全脱节 |

**检查点**:
- [ ] 至少 3 个角色在遮住角色名后，仅靠说话方式就能区分
- [ ] 关键场景（决策点前）的对白有足够信息量帮助玩家决策
- [ ] 不存在"为了解释而解释"的工具人台词
- [ ] 旁白/独白不过度使用，叙事优先通过对话推进
- [ ] 台词契合时代/世界观背景

---

### B2. 台词情感表达

**考核内容**: 台词是否能传递情感、引发共鸣？情绪层次是否丰富？

| 分数 | 标准 |
|------|------|
| 9-10 | 存在至少 2 个能引发强烈情绪反应的对话场景（紧张、感动、愤怒、幽默等）；情感通过台词节奏（长短、停顿、重复）和舞台指示配合呈现；情绪切换有过渡而非突变 |
| 7-8 | 有 1-2 个情感高潮场景；台词能准确传递预期情绪 |
| 5-6 | 台词情感平淡，缺乏高潮低谷；或情绪过度密集导致疲劳 |
| 3-4 | 台词与情境情绪不匹配（该悲伤时平淡，该紧张时松弛）|
| 1-2 | 全剧情绪单一，无任何情感起伏 |

**检查点**:
- [ ] 每集至少有 1 个明确的情绪节拍高点（舞台指示 + 台词配合传递）
- [ ] 幽默元素（如果有）不破坏整体氛围
- [ ] 紧张场景使用短句、断句；温情场景节奏放慢
- [ ] 角色的情绪状态在舞台指示中有明确标注（帮助演绎）

---

## C. 剧情与叙事 — 5 个维度

### C1. 分支剧情丰富度

**考核内容**: 改编后的分支叙事是否丰富多样？是否超越原作单线叙事？

| 分数 | 标准 |
|------|------|
| 9-10 | 分支不仅是"做了不同的事"，而是"进入了不同的故事线"；不同分支有独特的场景、角色互动、世界观展开；分支数量与时长匹配；存在延时分岔——早期选择在后续集数才显现后果 |
| 7-8 | 分支有实质差异；大部分分支有独立场景内容而非仅改几句台词 |
| 5-6 | 有分支但内容趋同（换个地点说差不多的话）；或分支过快汇合 |
| 3-4 | 分支是虚假选择——不同选项导向几乎相同的结果 |
| 1-2 | 无实质分支，所有选项殊途同归 |

**检查点**:
- [ ] 每个决策点的不同选项至少有 1 个独立场景（≥ 300 字差异化内容）
- [ ] 存在至少 1 条延时分岔——早期选择影响后续集数的剧情走向
- [ ] 分支不是简单的"好/坏"二元对立，而是"不同风格/策略"的展开
- [ ] 原作中的单一事件被改编出 ≥ 2 种有趣的可能性
- [ ] 分支质量一致性：各平行线（EP02/EP03/EP04）的内容质量、场景数量、台词密度应大致相当（偏差 ≤ 30%），不存在"充实主线 + 敷衍支线"的情况
- [ ] 延迟后果在后续集中有实际兑现

---

### C2. 情节完整性、连贯性与钩子密度

**考核内容**: 故事是否有头有尾？集与集之间过渡是否自然？每集末尾的剧情钩子是否足够？

| 分数 | 标准 |
|------|------|
| 9-10 | 每集有完整的小弧线（起承转合）；集末钩子强烈（让人想立即看下一集）；全剧有主控思想统领各集；无断裂感 |
| 7-8 | 每集有基本结构；大部分集末有钩子；偶有 1 集过渡略弱 |
| 5-6 | 部分集缺乏独立弧线（只是连续事件的切割）；钩子不均匀 |
| 3-4 | 多集结构松散；缺乏连接动力 |
| 1-2 | 剧情碎片化，无法辨识各集边界或整体走向 |

**检查点**:
- [ ] EP01 在前 2 个场景内建立核心悬念（主角的困境/目标）
- [ ] 每集最后一个场景包含悬念、反转或决策压力
- [ ] 全剧存在 ≥ 1 条贯穿线（未解之谜、长期目标、持续威胁）
- [ ] 集与集之间的时间/空间跳跃有过渡交代
- [ ] 钩子密度：每 8-10 分钟至少 1 个小钩子

---

### C3. 矛盾冲突与情节反转

**考核内容**: 核心矛盾是否清晰有力？反转是否有铺垫且令人意外？

| 分数 | 标准 |
|------|------|
| 9-10 | 存在 ≥ 2 层矛盾（外部冲突 + 内心冲突）；至少 1 个反转有"前面埋了伏笔但没注意到"的效果；冲突的升级是阶梯式的（每次解决一个问题后出现更大的问题）|
| 7-8 | 核心矛盾清晰；有 1-2 个有效反转；冲突有升级感 |
| 5-6 | 有矛盾但不够有力；反转缺乏铺垫或过于明显 |
| 3-4 | 冲突人为制造感强（"为了打而打"）；无有效反转 |
| 1-2 | 无明确矛盾驱动，剧情靠事件堆砌推进 |

**检查点**:
- [ ] 全剧有 1 个核心外部矛盾（主角 vs 什么）和 1 个内心矛盾（主角 vs 自己）
- [ ] 至少 1 个反转在之前有 ≥ 2 处伏笔/暗示
- [ ] 冲突不是靠"误会"驱动（除非是喜剧）
- [ ] 中段不出现"主角无敌碾压一切"的乏味局面
- [ ] 主角能力增长合理，中段有挫折

---

### C4. 情节逻辑性与分支闭环

**考核内容**: 事件因果链是否合理？分支是否有闭环（不留悬空线头）？

| 分数 | 标准 |
|------|------|
| 9-10 | 所有分支最终汇合或到达明确结局（无悬空）；事件因果链清晰可追溯；不依赖巧合推进关键剧情；跨分支的信息不矛盾 |
| 7-8 | 主线逻辑通顺；≤ 1 条分支有轻微悬空 |
| 5-6 | 部分事件因果链薄弱；1-2 条分支悬空 |
| 3-4 | 逻辑漏洞明显；多条分支悬空无交代 |
| 1-2 | 事件发展随意，因果关系混乱 |

**检查点**:
- [ ] 每个决策点的每个选项都有明确的跳转目标且目标场景存在
- [ ] 分支场景最终汇入后续主线或到达结局（死亡结局 / 正式结局）
- [ ] 不存在"提到但未兑现"的设定
- [ ] structure.json 中所有 leads_to 引用的场景在脚本中存在
- [ ] 同一时间线下，不同分支中的"公共事件"描述一致
- [ ] 不存在"凭空出现"的能力/武器——每个能力首次使用前有获取/学习场景
- [ ] 已销毁/遗失物品不得在后续使用
- [ ] 已死亡角色不得复活出场

---

### C5. 整体节奏安排与开局吸引力

**考核内容**: 全剧节奏是否合理？开局是否有足够抓力？

| 分数 | 标准 |
|------|------|
| 9-10 | EP01 前 2 分钟内建立紧迫感和好奇心；全剧张弛有度（高潮后有缓冲、铺垫后有爆发）；中段无明显拖沓；结局节奏加速推向高潮 |
| 7-8 | 开局有吸引力；整体节奏基本合理，偶有 1 集偏慢 |
| 5-6 | 开局平淡需要"忍过前 2 集"；或全程同一节奏无起伏 |
| 3-4 | 开局劝退；中段严重拖沓 |
| 1-2 | 无法辨识节奏，剧情流水账 |

**检查点**:
- [ ] EP01 开场场景有"异常"或"冲突"（不是平淡日常介绍）
- [ ] 第一个互动决策点在 EP01 的前半段出现
- [ ] 全剧的情绪强度曲线大致呈"W 形"或"锯齿上升"
- [ ] 每集时长分配合理（误差 ≤ 原定时长的 30%）
- [ ] 最终集的节奏比前面更紧凑

---

## D. 互动设计 — 6 个维度

### D1. 选项与剧情配合度

**考核内容**: 决策点出现的时机是否自然？是否融入剧情而非突兀插入？

| 分数 | 标准 |
|------|------|
| 9-10 | 选择精准出现在主角被逼到墙角的那一刻；观众不由自主想替主角做决定；选项是主角的不同行动方式；选择前有足够铺垫蓄压让玩家理解困境 |
| 7-8 | 大部分选择出现在蓄压高潮处；1-2 处时机略偏后但仍在紧张区间 |
| 5-6 | 选择经常出现在事件结束后，像是追加的"下一步"问题 |
| 3-4 | 选择与剧情脱节 |
| 1-2 | 选择完全不融入叙事 |

**检查点**:
- [ ] 选择出现在集内的蓄压高潮处，而非事件结束后的尾巴
- [ ] 选择的 question 描述的是主角"此刻"面对的困境，不是"接下来"的计划
- [ ] 选项是主角的不同行动方式，不是对未来计划的选择
- [ ] 选择之后每个选项有完整的结果展开（不是只有一句话跳转）
- [ ] 每个选择前的铺垫场景提供了做决策所需的信息
- [ ] 选择的问题文本契合当前剧情语境（不是通用的"你想怎么做？"）
- [ ] 倒计时设置合理（紧急场景短倒计时，深思场景长倒计时或无倒计时）

---

### D2. 选项意义性（两难抉择设计）

**考核内容**: 不同选项是否导向不同后果？是否构成真正的两难抉择——没有明显最优解，玩家必须权衡取舍？

| 分数 | 标准 |
|------|------|
| 9-10 | ≥ 80% 的选择构成两难抉择——每个选项都有代价，不存在"明显最优解"；选项之间体现不同价值观的碰撞（忠义 vs 生存、个人 vs 集体等），属于感性比较而非理性计算；至少 2 个选择影响后续多集走向；选项 outcome 清晰描述不同后果 |
| 7-8 | ≥ 60% 的选择有差异化后果且需要权衡；有 1 个以上跨集影响的选择 |
| 5-6 | 约一半选择需要权衡；另一半有明显"正确答案" |
| 3-4 | 大部分选择是虚假选择或霍布森选择（看似有选，实则只有一个合理选项）|
| 1-2 | 所有选择都无实质后果，或有明显送分选项 |

**检查点**:
- [ ] 0% 的选择存在重复选项 text（同一选择内两个选项文本完全相同）
- [ ] 0% 的选择存在重复选项 outcome（同一选择内两个选项后果完全相同）
- [ ] 每个选择的不同选项通往不同的集，且下一集内容有实质差异
- [ ] 存在至少 1 个"无法两全"的选择——选 A 得到某样东西但失去另一样
- [ ] 不存在"送分选项"——一个选项在所有维度上都优于其他选项
- [ ] 选项文本本身不暗示"这是正确答案"（避免诱导性措辞）
- [ ] Flavor 选择的 question 必须紧扣本集蓄压高潮（不使用通用模板）
- [ ] 每个选项都是观众"如果是我也想试试"的行动（代入感）
- [ ] 选项描述的是主角的行为方式（暴力/智取/隐忍），不是剧情走向

---

### D3. 玩家掌控感（能动性）

**考核内容**: 玩家是否感觉自己在"主导"故事而非"被主导"？是否体验到"自致的命运"——结局是自己选择的结果？

| 分数 | 标准 |
|------|------|
| 9-10 | 玩家的选择能被后续剧情"记住"并呼应（角色提到之前的决定）；选择频率适中（不过多也不过少）；不同选择路径的最终结局让玩家感觉"这是我自己走到这里的" |
| 7-8 | 选择有后续呼应；频率合理 |
| 5-6 | 选择后立即有反馈但后续不再提及；或选择间隔过长 |
| 3-4 | 剧情大量自动推进，选择像装饰 |
| 1-2 | 选择纯粹是形式，剧情走向完全不受影响 |

**检查点**:
- [ ] 每集 2-4 个决策点（不多不少）
- [ ] 选择后的下一个场景立即体现选择后果
- [ ] 后续集中至少有 1 处提及/呼应之前的选择（系统提示或角色对话）
- [ ] 不存在连续 3 个以上场景无任何互动机会
- [ ] 隐藏要素让玩家感觉"选择有深远影响"
- [ ] 每个选择的不同选项在后续有差异化内容

---

### D4. 结局差异化与重玩吸引力

**考核内容**: 结局是否真正不同？是否值得重玩探索？行业标杆：8 集作品应有 ≥ 4 个差异化结局 + ≥ 2 个死亡结局，总内容量为单次通关的 2.5 倍以上。

| 分数 | 标准 |
|------|------|
| 9-10 | ≥ 4 个差异化结局（不是同一结局的轻微变体）；结局反映玩家全程选择的累积效果；存在隐藏结局或特殊条件结局；看完一个结局后有强烈欲望去探索其他结局；不同路线的独占内容足够丰富（单次通关只能看到 ~30-40% 的总内容）|
| 7-8 | ≥ 3 个差异化结局；结局与选择路径有明确关联；有一定重玩内容 |
| 5-6 | 有多个结局但差异仅限于最后 1 集的变化 |
| 3-4 | 结局只有好/坏两种，缺乏层次 |
| 1-2 | 只有 1 个结局或结局间无差异 |

**检查点**:
- [ ] 正式结局数 ≥ 4，且至少覆盖 3 种不同的情感基调（欢喜 / 苦乐参半 / 悲剧 / 开放）
- [ ] 死亡结局数 ≥ 2，每个有 ≥ 200 字独特描写
- [ ] 至少 1 个隐藏结局或需要特殊条件（如累积变量达到阈值）才能触发的结局
- [ ] 结局摘要中能明确看出"哪些选择路径导向哪个结局"
- [ ] 总脚本内容量（所有集所有分支的总字数）≥ 单次通关路径字数的 2.5 倍

---

### D5. 玩法深度与叙事后果

**考核内容**: 在基本互动之上，是否有跨集延迟后果、条件解锁等设计提升可玩性？选择是否通过叙事后果（而非数值）产生真正的影响？

| 分数 | 标准 |
|------|------|
| 9-10 | ≥ 3 个选择的后果在后续集中可见呼应（角色提及、情节变化）；≥ 2 个延时分岔——早期选择在后续集产生实质不同剧情；不同路线有独特的角色关系发展和场景内容；存在 QTE 或轻量玩法标记点（可选加分）|
| 7-8 | ≥ 2 个选择有跨集后果呼应；≥ 1 个延时分岔；路线之间有明显差异 |
| 5-6 | 选择有即时后果但后续不再提及；路线差异仅限于几句台词变化 |
| 3-4 | 选择无后续影响，所有路线实质相同 |
| 1-2 | 无任何延迟后果或路线差异 |

**检查点**:
- [ ] 至少 2 个选择的后果在 ≥ 2 集后有可见呼应（角色台词或情节变化）
- [ ] 至少 1 个延时分岔——早期选择导致后续集出现额外选项或不同剧情
- [ ] 不同分支路线有独占场景内容（不只是换台词）
- [ ] 选择后果在选项 `outcome` 字段中有清晰描述
- [ ] （加分）标记 ≥ 1 个 QTE 或轻量玩法点位（格式：`【QTE 标记】[描述]`）

### D6. 试错空间

**考核内容**: 选择是否允许犯错？犯错后是否有有趣的体验而非简单惩罚？

| 分数 | 标准 |
|------|------|
| 9-10 | "错误"选项不是简单的死亡结局，而是进入有趣的替代剧情；死亡结局有完整叙事而非突然终止；存在"看起来是坏选择但其实有隐藏好处"的选项；失败路径也有独特体验值得探索 |
| 7-8 | 错误选项有一定叙事内容；死亡结局有简短但有质感的结束场景 |
| 5-6 | 错误选项导致直接死亡结局但有短暂提示 |
| 3-4 | 错误即死，无任何缓冲或叙事 |
| 1-2 | 无错误选项或所有选项等价 |

**检查点**:
- [ ] 死亡结局场景有 ≥ 200 字的专属剧情描写
- [ ] 至少 1 个"看起来冒险"的选项提供了独特的（好的或坏的）体验
- [ ] 非死亡结局的"次优选择"让玩家走更难的路但不直接失败
- [ ] 全剧死亡结局数量合理（8 集建议 1-3 个）
- [ ] 失败后有"提示"让玩家理解为什么失败（不是随机死亡）

## E. 结构硬性要求（通过/不通过）

以下不参与评分，但任何一项不通过则整体不及格。

### E1. 分支收束结构

全剧必须采用**多重分叉收束**结构，分叉数量与集数匹配：

| 总集数 | 最少3路分叉 | 最少2路分叉 | 最少总分叉数 |
|---|---|---|---|
| 6-8 | 1 | 0 | 1 |
| 9-12 | 1 | 2 | 3 |
| 13+ | 1 | 3 | 4 |

示例（10集结构）：
```
EP01（共同开局）
  ├─A→ EP02a ─┐
  ├─B→ EP02b ─┼→ EP03（汇聚）
  └─C→ EP02c ─┘
                 ├─A→ EP04a ─┐
                 └─B→ EP04b ─┘→ EP05（汇聚）
                                  ├─A→ EP06a
                                  └─B→ EP06b
```

**要求**:
- [ ] EP01 为共同开局，末尾有第一个分叉点
- [ ] 至少 1 个 3 路分叉 + 满足复杂度表的 2 路分叉数量
- [ ] 每个选择的每个选项通往不同的集（菱形结构）
- [ ] 菱形分支经 1-2 集后汇聚回同一集
- [ ] 死胡同集散布全剧（总集数的 15-25%），不集中在开头或结尾
- [ ] 终局分支各自到达不同结局
- [ ] 单次通关路径集数 = `round(episodes * playthrough_ratio)`

### E2. 延时分岔与条件解锁

早期选择必须在后续集产生可见的后果：

- [ ] 至少 1 个前置条件解锁：早期某个选择导致后续某集出现额外选项（选过 A 的玩家有 3 个选项，没选过的只有 2 个）
- [ ] 至少 1 个条件触发差异：后续某个相同的决策点，因为早期选择不同，导致相同选项产生不同结果（如：早期得罪了某人，后来求助时被拒绝；早期帮了某人，后来求助时被接受）
- [ ] 这些延时分岔在脚本中有明确体现

### E3. 时长合规

- [ ] 每集时长在 `min_duration / 总集数` 到 `max_duration / 总集数` 之间
- [ ] 总时长（所有集的 duration_minutes 之和）在 min_duration 到 max_duration 之间

### E4. 内容体量

总脚本内容量（所有集所有分支的总字数）必须达到单次通关路径字数的 **≥ 2.5 倍**。

- [ ] 计算单次最短通关路径的总字数 S（通关集数 = round(episodes × playthrough_ratio)）
- [ ] 计算全部脚本（含所有分支、死亡结局）的总字数 T
- [ ] T / S ≥ 1 / playthrough_ratio（例如 ratio=0.6 时，T/S ≥ 1.67；ratio=0.4 时 T/S ≥ 2.5）
- [ ] 各分叉线程的独占内容各不少于 S × 0.1

### E5. 决策密度

决策点分布必须匹配行业节奏标准：每 3-5 分钟出现一个有意义的决策点。

- [ ] 计算每集的决策密度：决策点数 / 时长分钟数
- [ ] 每集决策密度在 0.2 ~ 0.33 之间（即每 3-5 分钟 1 个决策点）
- [ ] 不存在连续 ≥ 6 分钟无任何互动机会的"纯看片"段落
- [ ] 不存在连续 2 分钟内出现 ≥ 2 个决策点的"决策轰炸"

### E6. 因果完整性

全剧必须满足能力来源可追溯、集间过渡一致等因果要求：

- [ ] 每个能力/武器/道具在脚本中首次使用前有获取/学习场景
- [ ] 相邻集的地点、时间、状态逻辑连续（无"瞬移"、无"复活"、无"失忆"）
- [ ] 已死亡角色在死亡集之后不得出场
- [ ] 已销毁/遗失物品在之后不得再使用
- [ ] 新角色首次出场有介绍场景

### E7. 爽点密度

每集至少 1 个明确的爽点时刻（满足感/成就感/反转快感的高潮），匹配题材特性：

- [ ] `satisfaction_beats` 中每集至少有 1 个条目
- [ ] 爽点出现在选择之后的分支结果中——不同选项 = 不同的爽点释放方式
- [ ] 爽点有铺垫——对应的 `setup_scenes` 在前面存在压力/困境积累
- [ ] 爽点类型在各集间有变化（不全是同一种类型）
- [ ] 关键对话有可看性：存在反转打脸、信息差利用、潜台词层等技巧

**题材常见爽点参考**：

| 题材 | 典型爽点 |
|------|----------|
| 大女主古言 | 怼人打脸、零元购搬空、揭马甲、护短、断亲宣言、虐渣、白莲花翻车、囤货炫富、降维打击、男主折服 |
| 修仙 | 升级突破、打败挑衅者、获得宝物功法、扮猪吃虎、逆天改命 |
| 末世 | 绝境反杀、发现安全区、获得稀缺资源、揭露真相、牺牲换胜利 |
| 都市 | 打脸装逼、身份揭露、商战翻盘、复仇成功、真相大白 |
| 悬疑 | 推理破局、反转揭秘、伏笔回收、真凶意外、密室逃脱 |
| 电竞 | 绝杀翻盘、天才操作、团队配合、以弱胜强、宿敌对决 |
| 宫斗 | 反将一军、揭穿阴谋、隐忍后爆发、借刀杀人、联盟逆转 |

**代入感检查（女频题材必检）**：

- [ ] 每个爽点的铺垫是否让观众感到"替主角委屈"——虐的程度足够具体、不公的点足够清晰
- [ ] 每个爽点的释放是否让观众感到"替主角出气"——反击彻底、不半途而废
- [ ] 旁观者反应是否充当了情绪放大器——围观者的震惊/哑口无言/后悔是否到位
- [ ] 铺垫的"虐"是否层层递进——后面的欺辱比前面更过分，对应的反击也更爽
- [ ] 选择是否反映了观众真正想做的事——"如果是我，我也想这么干"

### E8. 脚本格式合规

脚本必须遵循专业影视剧本格式规范（参照《天坑追匪》），确保可直接用于拍摄/制作：

- [ ] 每个场景以标准两行表头开头：`场：NNN 景：场景名` / `时：时间 人：角色列表`
- [ ] 场号全剧连续编号（001, 002, ...），跨集不重置
- [ ] 所有 `▲` 舞台指示使用自然影视镜头语言，镜头提示以括号嵌入（如 `（特写）`、`（手持）`、`（主观）`），不使用固定 【镜头】 标签
- [ ] 内心独白通过表演指示自然呈现（`（喃喃自语）`、`（心想）`、`（暗忖）`），不使用 `（内心独白）` 或 `（os）` 标记
- [ ] 选项文本 ≤ 8 个中文字符，选择格式为 `选择 NNN` + `问题：xxx` + `NNNA：选项文本`
- [ ] **每个 `选择 NNN` 之后必须有 `问题：` 行**（描述主角此刻面对的困境，让玩家理解选择背景）
- [ ] 分支内容使用 `━━━━━━━━━━ NNNA：xxx ━━━━━━━━━━` 分节格式，禁止并列列格式
- [ ] 选项文本不包含模板标记（如 `（≤8字）`）
- [ ] 场景过渡明确标注跳转：`NNN 场结束，接 NNN 场` 或 `选择 NNNA 结束，接第N集 NNN 场`
- [ ] 条件路由使用 `判定` 格式，死亡结局使用 `BE：描述` 格式
- [ ] 新角色首次出场有字幕条标注（`（角色名字幕条）` 或 `人名字幕条：角色名，身份`）

### E9. 选择质量

每集选择必须有差异化选项，禁止假选择（相同文本或相同后果）：

- [ ] 每集选择至少 2 个选项
- [ ] 不存在任何两个选项 text 完全相同
- [ ] 不存在任何两个选项 outcome 完全相同
- [ ] question 不使用通用模板（"接下来怎么办"、"你的选择是"、"怎么办"等）
- [ ] 每个选择的每个选项 next 指向不同的集（铁律：无例外）
- [ ] 死胡同集散布全剧，每 3-5 个正常集之间至少出现 1 个死胡同选项
- [ ] **脚本中每个 `选择 NNN` 之后有 `问题：` 行**（缺失 = 不合规）
- [ ] 问题行描述主角此刻面对的具体困境（而非通用描述）

## 评估流程

### 第一轮：结构校验（硬性）

对照脚本、structure.json、story_bible.json 做数据校验：
1. 所有场景编号双向匹配（脚本 ↔ structure）
2. 所有决策点编号、选项标签、跳转目标匹配
3. 所有死亡结局标记匹配

6. 结构硬性要求（E1/E2/E3/E4/E5/E6/E7/E8/E9）检查

### 第二轮：内容评估（逐维度打分）

阅读完整脚本后，逐一对 16 个维度打分（1-10），并记录扣分理由和具体场景引用。

### 第三轮：综合评审

1. 汇总分数，检查是否有维度 ≤ 3（硬性不及格线）
2. 检查总分是否 ≥ 112 / 160
3. 列出前 3 优点和前 3 待改进项
4. 给出最终评级

| 评级 | 总分 | 说明 |
|------|------|------|
| S | ≥ 144（90%）| 卓越，可直接投入制作 |
| A | ≥ 128（80%）| 优秀，小修后可投入制作 |
| B | ≥ 112（70%）| 良好，需定向修改 |
| C | ≥ 96（60%）| 及格，需较大幅度修改 |
| D | < 96 | 不及格，建议重写 |
