## Phase 4.5 轻量节点校验

- 你只校验 `current_node`。
- `reader_has_seen` 是 DFS 后的压缩读者记忆，不是完整图。
- `content_text` 是读者实际看到的节点文本，可能被截断。

- 不要重新检查确定性规则：二叉出边、无环、schema、可达性、场景连续性、结局数量都已由代码检查。
- 只报告能用一个节点的 prose、skeleton 节拍、question、choice label、produces/requires 修复的问题。

## 你检查的内容

- **S1 — 选择实质性与吸引力**（细则见 CHOICE_DESIGN.md R1-R13）
  - 每个选择是否通往实质不同的体验？
  - **人格投票测试（R5）**：爽感型/情感型/谨慎型三种玩家，每个选项必须至少是
    一种人格的首选。任一选项三票全无 → 违规（被支配选项/没人会选）。
  - **收益审计（R2/R3）**：每个选项的卖点必须写在文本里（label/resolution/问题），
    且两边具体度对称——一边"立刻赢"另一边"或许有用"= 违规。
  - **赌注层级（R1/R8）**：问题必须挂在场景最高层级目标上（生死>使命>阵营>关系>脸面）；
    高潮节点的两难必须是同层级价值对撞。手握叛国铁证却把口角当并列选项 = 违规。
  - **确定性对称（R4）**：一边确定收益 vs 一边模糊收益 = 倾斜，违规。
  - 标记：伪选择（角色不可能做出）、不合逻辑的选择、倒退选择（不推进故事）
  - **同目标双选择是合法设计**（"选择写状态"模式）：两个选择指向同一目标节点时，
    只要各自的 `state_delta` 不同，就不是伪选择；此时检查两选项是否表达真正不同的
    行事方式与代价。

- **S2 — 文字忠实度**
  - prose 是否预设了 `requires` 中未声明的前置内容？
  - prose 是否断言了后续节点依赖但 `produces` 中未声明的新事实？
  - 隐含回指（"还带着她手上的温度"、"同一个人"、"再次"）未经声明 → **最高优先级**
  - 普通人物/物件首次出场不是违规 — 应在 prose 中自然引入

- **S3 — 冷引入优雅度**
  - 首次出现的元素读起来是优雅的惊喜还是突兀的？
  - 只标记：改变因果、解决冲突、成为后续依赖的关键物件/能力/事件
  - 不标记：普通角色出场、装饰物、一次性动作

- **S4 — 标签可读性**
  - 每个选择标签（≤8 中文字）是否清晰、以玩家视角书写、不泄露目标内容？
  - 标签是否为**具体动作**（对象+手段）？抽象策略范畴词（如「以智取胜」型）→ 违规
  - question 与 labels 的用词是否一致？question 说法 A、label 说法 B（同义不同词）→ 违规

- **S6 — 转场连贯（剧情层面）**
  - `reader_has_seen`/上一场 exit_context 与本场开场之间，剧情是否连贯？
  - 地点变化本身**不是**问题（转场切地点是正常电影语法）
  - 违规仅限：时间倒退、同场景内无法解释的瞬移、因果断裂（上一场的悬置动作凭空消失）

- **S5 — 语气与风格**
  - 是否与故事圣经的格调和原著语气一致？
  - `DEAD_END` 节点 prose 末尾的 `BE` 是合法结构标记，不是风格错误

- **S8 — 旁白职能**
  - 旁白（`旁白：`/narration）只能承载两类内容：①背景/底色 ②人物内心想法。
  - **违规**：任一 narration 元素承载了**关键事件**——剧情转折、冲突的发生或结果、关键动作、
    重大揭露、关键物件/证据的出现或易手——而不是用 action+dialogue 演出来。
  - 判断标准：删掉这条旁白后，观众是否就不知道"发生了什么大事"？是 → 它复述了关键事件 → 违规（severity=high）。
  - 不标记：背景交代、人物心理、情绪渲染、至多 1-2 句的 prologue 定场旁白。

- **S9 — 开场锚定**（仅对 prologue/根节点）
  - 根节点必须演出**原著字面意义上的第一场戏**（故事真正的起点），不是后续才到达的高潮或危机。
    若根节点直接落在后段危机、把真正的开场跳过 → 违规（severity=high）。
  - 根节点的**身份/前史/能力来源**必须靠开场的现场动作与对白演出，**不得**用旁白成段告知。
    出现「旁白：他/她其实另有身份、来历……」式成段交代身份或前史的补叙 → 违规（severity=high）。
  - 不标记：至多 1-2 句仅交代时代/地点的定场旁白。

- **S7 — 决断点在场（staging）**
  - question 点名的处境（地点、物件、对手、时间压力）是否在 prose 的**结尾段实际在场**？
    问题问「引爆暗锁还是巧解暗格」而 prose 从未抵达那道锁前 → 违规（凭空提问）。
  - 任一选项的标志性手段是否已在 prose 正文中被执行或演示（哪怕用在别的物件上）？
    是 → 违规（选项被预演，玩家的选择失去意义）。
  - prose 必须以两难局面收束：两条路的工具/路径/代价对玩家可见，但都未迈出。

## 规则

- 发现要**具体、可操作** — 指明节点、事实、修复方案
- 不要复述 prose 内容
- 不要发明新的结构性规则
- 不属于 S1-S9 的问题用 `check: "other"` 报告
- 如果问题需要完整图才能判断，返回通过，不要猜测

## 修复建议规则

- 首次出现太突兀 → 建议在 prose 中自然引入或移除
- 后续依赖的新事实 → 建议添加 `produces` + `new_facts`
- 只有节点确实无法在没有先前状态下成立时才建议添加 `requires`
- 不要建议 `char.<name>.present` 来修复普通角色出场
